## train_start

Start a LoRA training job: target 'local' builds the config and launches the GPU trainer container (docker run --gpus all); target 'pod' ssh-drives pod-native training on a connected RunPod pod (pod_id, or the connector's currently connected pod). Returns a job id for train_status/train_cancel. Long-running — returns immediately; poll train_status. On completion the LoRA is delivered per deliverTo (pod/local/both) and cataloged when local. Run train_doctor first if unsure the image/docker/GPU (local) or bootstrap (pod) are ready.

### Parameters

<ParamField path="name" type="string" required>
  Job name — becomes the output .safetensors basename (e.g. 'aria_character').
</ParamField>
<ParamField path="flow" type="enum" required default="character">
  Options: `character`.
</ParamField>
<ParamField path="model" type="enum" required default="flux1-dev">
  Options: `flux1-dev`.
</ParamField>
<ParamField path="datasetPath" type="string" required>
  Dataset dir from train_prepare_dataset (images + same-basename .txt captions).
</ParamField>
<ParamField path="trigger" type="string">
  Unique trigger word (e.g. 'ohwx person') — injected as trigger_word and usable in prompts.
</ParamField>
<ParamField path="params" type="object">
  —
</ParamField>
<ParamField path="device" type="string">
  GPU selector, default cuda:0.
</ParamField>
<ParamField path="target" type="enum" required default="local">
  'local' = docker on this rig; 'pod' = pod-native over ssh on a RunPod pod. Options: `local`, `pod`.
</ParamField>
<ParamField path="pod_id" type="string">
  RunPod pod to train on (target 'pod'). Default: the connector's currently connected/watched pod.
</ParamField>
<ParamField path="deliverTo" type="enum" required default="both">
  Pod jobs only: where the finished LoRA lands. Options: `pod`, `local`, `both`.
</ParamField>
<ParamField path="model_path" type="string">
  Override the base model path AS THE TRAINER SEES IT (pod path for target 'pod', container path for 'local') — e.g. a pre-uploaded local HF snapshot dir when the default HF repo id is gated/unreachable.
</ParamField>

## train_cancel

Stop a running training job (docker stop) and mark it cancelled. Checkpoints already saved stay in the job's output dir; no LoRA is handed off to models/loras. Returns ok:false when the container could not be confirmed stopped (the job reverts to running).

### Parameters

<ParamField path="id" type="string" required>
  Job id from train_start.
</ParamField>
